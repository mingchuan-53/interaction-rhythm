看见你的数字手感。

**扣舷是一个本地优先的 Windows 桌面小工具，用来观察键盘响应、鼠标响应和前台应用强度。**

它不判断你是不是高效，也不记录你输入了什么。它只把电脑前的一天整理成更容易回看的线索：什么时候最活跃，哪个应用成为主场，哪些时段像键盘冲刺，哪些应用只是安静驻留。

“扣舷”取意自“扣舷而歌”：不窥探内容，只听见动作留下的轻响。这个工具想做的不是监督，而是把你的数字生活变成有形状、有强度、可回看的手感画像。

> 当前仓库只包含 Windows 桌面端。Android 版仍在单独实验中，不随本仓库发布。

## 安装

下载最新版本：

[→ Releases 页面](https://github.com/mingchuan-53/interaction-rhythm/releases/latest)

- **Windows 安装器**：`interaction-rhythm-setup-v版本号.exe`。适合普通用户，带开始菜单、桌面快捷方式和卸载入口。
- **Windows 便携包**：`interaction-rhythm.zip`。解压后运行 `InteractionRhythm.exe`，适合朋友测试、临时试用和保留多个版本。
- **中文分享包**：本地构建时会生成 `扣舷.zip`，主要用于中文朋友测试。GitHub Release 使用稳定英文文件名。

扣舷会在启动后轻量检查 stable 发布通道。有新版时只提示，不会自动安装；安装必须由用户确认。更新包下载和校验后，应用会短暂关闭、替换程序文件、恢复用户数据，再自动重新打开。

## 它在你桌面系统上的痕迹

扣舷写入的位置：

- `data/tracker.db`：本地 SQLite 数据库，保存键盘响应次数、鼠标响应次数、前台应用停留和单应用交互归属。
- `data/icons/`：应用图标缓存，只来自本机可执行文件路径。
- `data/settings.json`：用户设置，例如自动检查更新、开机自启、后台启动等。
- `data/updates/`：更新包暂存、更新日志和最近一次更新阶段状态。
- `data/diagnostics/`：本地诊断日志，用来排查启动、托盘、窗口和更新问题。

**扣舷默认不会把数据上传到服务器。** 只有当你点击 AI 图标生成 MiMo 手感回放时，它会发送当天的聚合统计，例如键盘次数、鼠标次数、小时峰值和应用名称；不会发送具体按键、输入文本、鼠标坐标、屏幕截图或窗口正文。

## 它能做什么

### 看见每天的数字手感

- 统计键盘响应和鼠标响应次数。
- 最近 6 天小时热力图，可切换键盘、鼠标和总响应视角。
- 热力图只呈现强度，不呈现内容。

### 看见应用停留和交互强度

- 前台应用强度榜。
- 单应用分析：停留时长、手感响应、手感密度、小时分布和会话记录。
- 对照“停留时间”和“键鼠响应”，区分安静驻留、键盘冲刺、浏览筛选和碎片回返。

### 用手感回放做轻量观察

- 点击 AI 图标后调用 MiMo 生成当天手感回放。
- 只发送聚合统计，不发送输入内容、窗口正文、截图或鼠标坐标。
- 不做人格判断，不给效率打分，不假装知道你的真实意图。

### 托盘后台运行

- 关闭窗口默认隐藏到托盘。
- 托盘里可以打开窗口、刷新数据、退出后台。
- 完全退出必须从托盘菜单执行，避免误关后丢失连续记录。

### 导出自己的数据

- 支持 JSON 和 CSV。
- JSON 适合继续归档或交给其他工具分析。
- CSV 适合放进表格里查看。

### 准备媒体素材

- 设置页可以生成选中日期的媒体素材快照。
- 快照会整理成可复制的 Markdown 草稿，方便继续做图文、短视频脚本或发布说明。
- 快照只使用本机聚合统计和手感回放结果，不包含输入内容、具体按键、鼠标坐标、截图或窗口正文。

## 隐私边界

扣舷会记录：

- 每批键盘响应次数。
- 每批鼠标响应次数。
- 前台应用名称、窗口标题、应用路径。
- 应用前台停留的开始和结束时间。
- 键盘 / 鼠标响应对应的前台应用。

扣舷不会记录：

- 具体按下了哪个键。
- 输入的文字内容。
- 鼠标坐标或屏幕截图。
- 浏览器页面正文。
- 后台上传数据。

窗口标题可能包含网页标题、文档标题或聊天窗口名称。公开截图前请使用测试数据，或者清理私人信息。

## 从源码运行

环境要求：

- Windows 10 或更高版本。
- Python 3.10 或更高版本。

安装依赖：

```powershell
pip install -r requirements.txt
```

启动：

```powershell
python main.py
```

启动后会打开无边框桌面窗口，并在系统托盘保留入口。本地 HTTP 服务默认监听：

```text
http://127.0.0.1:18923
```

## 打包

Windows 便携包：

```powershell
.\build.ps1
```

Windows 安装器：

```powershell
.\build-installer.ps1
```

构建产物：

- `dist/current/InteractionRhythm/`：本机当前运行版。
- `dist/releases/interaction-rhythm.zip`：GitHub Release 使用的便携包。
- `dist/releases/扣舷.zip`：中文朋友测试包。
- `dist/releases/update.json`：应用内更新清单。
- `installer/output/interaction-rhythm-setup-v版本号.exe`：Windows 安装器。

发布包生成后会运行预检，阻止数据库、日志、缓存和调试截图进入分享包。

多平台安装包策略见 [docs/packaging/installers.md](docs/packaging/installers.md)。

## 架构

| 层 | 路径 | 作用 |
| --- | --- | --- |
| 应用入口 | `main.py` | 单实例、启动流程、托盘、窗口和关闭更新流程。 |
| 数据层 | `db.py` | SQLite 表结构、统计、导出和手感回放规则。 |
| 记录层 | `monitor.py` | 键盘响应、鼠标响应和前台窗口停留记录。 |
| 本地接口 | `stats.py` | HTTP API、静态页面、导出、设置和更新接口。 |
| 桌面壳 | `window.py`、`tray.py` | pywebview 窗口、无边框拖动、系统托盘。 |
| 更新 | `update_manager.py` | 检查更新、下载、校验、备份、替换和回滚。 |
| 前端 | `static/index.html` | 主界面、弹窗、热力图、应用手感构成、设置和手感回放。 |

## 内部工作机制

键盘和鼠标只以“次数”进入数据库。前台应用由窗口监控轮询记录，应用切换时写入上一段使用时间。键鼠响应会归属到当时前台应用，因此单应用分析可以比较“这个应用停留了多久”和“你在它上面实际操作了多少”。扣舷把这些信号称为数字手感，而不是效率分数。

更新走下载、校验、暂存、备份、替换、恢复数据、重启的流程。用户数据目录不会被更新包覆盖；更新阶段会写入本地状态文件，失败时优先恢复旧程序和用户数据。
