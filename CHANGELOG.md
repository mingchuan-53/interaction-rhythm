# 开发日志

这个日志记录叩舷从 Type Tracker、交互节律到当前版本的主要产品变化。它不是逐条提交记录，而是按用户体验和技术方向整理的开发脉络。

## 1.9.7

定位：AI 默认可用版。让手感回放不再依赖用户进入界面配置模型或 Key。

- AI 回放新增通用 provider 选择层，前端仍走同一个入口，后端自动选择可用模型。
- 默认优先调用本机 Ollama 的 `gemma4:12b`，本地可用时记录不会离开本机。
- 支持通过 `KOUXIAN_AI_PROVIDER`、`KOUXIAN_AI_API_KEY`、`DEEPSEEK_API_KEY`、`KOUXIAN_AI_BASE_URL` 和 `KOUXIAN_AI_MODEL` 接入 DeepSeek 或其他 OpenAI 兼容服务。
- AI 弹窗文案从 MiMo 专名改为通用 AI，并展示后端返回的实际隐私说明。
- 对 thinking 模型返回空 `content` 的情况做兼容，减少“分析成功但页面无内容”的失败感。

## 1.9.6

定位：媒体素材快照版。为后续制作图文、短视频和发布说明准备一份可复制的当天手感素材。

- 设置页新增“媒体素材”入口，可按当前选中日期生成 Markdown 快照草稿。
- 新增本地 `/api/media-snapshot` 接口，复用当天聚合统计、应用强度和手感回放结论，不扩大记录范围。
- 素材草稿包含手感类型、键盘 / 鼠标响应、活跃时长、峰值时段、主场应用、可讲线索和隐私边界。
- 空数据日也会生成可用草稿，避免制作内容时遇到空状态断层。
- 设置页把外观放到最上方，并按“明暗模式 → 主题色 → 语言”的顺序组织，避免弹窗出现滚动条。
- 热力图新增年视图，由 12 张月视图组成；周 / 月 / 年切换迁移到底部左侧。
- 热力图色阶图例迁移到底部右侧，并增加浅色细分，让低强度日更容易辨认。

## 1.9.1

定位：P0 修复版。修掉 1.9.0 中顶部信息过密、应用榜边框过重、AI 入口仍像本地规则的问题。

- 顶部日期看板删除解释性摘要和右侧“响应”标签，只保留日期、数字手感和总响应数字。
- “键盘敲击”统一改为“键盘”，热力图标题里的“键盘鼠标”合并显示，不再留空隙。
- 应用强度榜改为无单项边框的双列列表，减少卡片套卡片感。
- AI 图标入口改为调用真实 MiMo，使用本机聚合统计生成当天手感回放；失败时明确显示错误，不用本地规则冒充。
- 回放文案统一使用“键盘响应 / 键盘冲刺”，避免旧的“敲击”术语回流到界面。
- README 更新隐私边界：默认本地，只有点击 MiMo 回放时发送聚合统计，不发送输入内容或窗口正文。

## 1.9.0

定位：递还今天稳定版。产品显示名统一为“扣舷”，把首页、热力图、应用构成和手感回放都收束到“这一天的数字手感”。

- 产品显示名从“叩舷”调整为“扣舷”，同时保留旧启动项、旧图标和旧数据路径兼容。
- 图标方向统一为“舷壁 + 扣痕 + 余响”，同步应用内图标、托盘图标和发布包资源。
- 首页增加日期看板，可查看前一天、后一天并回到今天；数字手感、热力图和应用手感构成都跟随日期变化。
- 首页摘要升级为“今天的形状”，优先呈现主场应用、键鼠偏向和峰值小时。
- 热力图改为一天一列的小时视图，并增加月视图；点击小时格可以打开这一小时的应用构成。
- “应用强度榜”改为“应用手感构成”，排序优先看键鼠响应和手感密度，而不是只看停留时长。
- 手感回放默认讲选中日期，最近 7 天回放保留为次级入口；回放文案弱化命令感和诊断感。
- 首屏数据加载改为分区成功即可显示，今日数据、热力图、应用构成任一接口失败时不会拖垮整页。
- 应用图标提取增加失败负缓存，减少坏路径反复提取导致的闪烁。
- 发布预检扩展到 release staging 目录，防止数据库、日志、图标缓存和调试截图进入可分享目录或 zip。

## 1.8.0

定位：叩舷大更新。从“交互节律”收束为“叩舷：看见你的数字手感”，把产品重心从严肃节律观察转向应用强度画像、趣味回放和可长期继承的个人数据资产。

- 产品显示名改为“叩舷”，副标题改为“看见你的数字手感”；仓库名、可执行文件名和更新包英文名暂时保持兼容。
- 主界面改为浅色个人统计仪表盘，使用柔和白色卡片、玫红强度视觉、胶囊切换和更清晰的数字层级。
- 默认主题色改为传统绿色，并在设置中加入主题色与中英文切换。
- 首页改为单页仪表盘，应用强度榜扩展为十项双列排行，不再需要滚动查看。
- 启动时会自动扫描并合并旧安装目录、旧 release 和 archive 里的 `tracker.db`，兼容远古版本数据继承。

- “节律助手”改为“手感回放”，输出主场应用、敲击冲刺、碎片回返、夜航时段等更具趣味性的观察线索。

- 继续沿用旧版 `data/tracker.db`、`settings.json` 和 `InteractionRhythm.exe` 更新链路，升级后继承之前版本记录。

## 1.7.2 准备中

定位：维护版，先补诊断、备份可追踪性和打包前检查，不扩展记录范围。

- 新增本地诊断日志，启动、关闭、托盘、窗口、更新检查、下载、校验和安装准备失败会写入 `data/diagnostics/`。

## 1.7.1

定位：修复更新、设置和节律助手体验里的阻塞感与数据风险。

- 检查更新改为“本地缓存状态 + 后台静默确认”，打开更新弹窗时不再同步等待网络。

- 节律助手建议从四条收敛为三条，并按建议类型去重，减少重复进程名建议。

- 设置、更新、导出等弹窗增加轻量弹出和收起动画。
- 修正升级后本地更新缓存短暂显示旧版本号的问题。
