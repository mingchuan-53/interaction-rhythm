# 开发日志

这个日志记录交互节律从 1.0 到当前版本的主要产品变化。它不是逐条提交记录，而是按用户体验和技术方向整理的开发脉络。

## 1.7

定位：修复更新、设置和节律助手体验里的阻塞感与数据风险。

- 检查更新改为“本地缓存状态 + 后台静默确认”，打开更新弹窗时不再同步等待网络。
- 设置页先用内存状态立即渲染，再后台刷新设置，减少“正在读取设置”的停顿。
- 更新脚本在安装前备份用户数据，只覆盖程序文件和必要资源，避免升级后显示重新计数。
- 更新退出前执行数据库 checkpoint，降低 WAL 未落盘造成的显示缺口。
- 节律助手建议从四条收敛为三条，并按建议类型去重，减少重复进程名建议。
- 节律助手不再把交互节律自身作为单应用强度建议对象。
- 设置、更新、导出等弹窗增加轻量弹出和收起动画。

## 1.6

定位：从公开测试便携包推进到更稳定的普通用户分发。

- 启动后支持轻量自动检查更新，有新版时显示小提示，不自动安装。
- 设置页增加“自动检查更新”开关。
- 默认更新通道指向 GitHub Release，降低发布时漏配清单的风险。
- 打包脚本同时生成中文分享包 `交互节律.zip` 和 GitHub 英文包 `interaction-rhythm.zip`。
- 打包脚本会在替换 `dist/current` 前停止运行中的旧版本，减少文件占用导致的构建失败。
- 新增 Inno Setup 安装器脚本，可生成开始菜单、桌面快捷方式和卸载入口。
- 新增 `build-installer.ps1`，用于构建 `interaction-rhythm-setup-v版本号.exe`。
- 已用独立 `v1.5` 测试副本验证端到端更新：发现 `v1.6`、下载安装、替换、重启后显示已是最新版本。

## 1.5

定位：从“统计展示”推进到“节律解释”。

- 新增单应用键盘、鼠标响应归属，键鼠次数会记录到当时前台应用。
- 单应用分析增加响应次数、响应密度和交互来源。
- 导出数据增加 `interaction` 记录，方便后续用表格或其它工具分析。
- 节律助手改为“结论 / 重点发现 / 建议”，不再展示完整分析过程。
- 节律助手开始对照应用时长占比与响应占比，识别停留、密集操作、回返等线索。
- 补齐开源 README、许可、贡献说明和版本日志。

## 1.4

定位：把设置、更新和 AI 弹窗收束成更完整的桌面产品形态。

- 设置页改为一屏展示，减少滚动。
- 增加静默启动选项。
- 开机自启、后台启动、导出数据、检查更新、切换主题归入设置入口。
- 优化节律助手排版，减少说明性文字。
- 优化检查更新弹窗，关闭后回到设置页。
- 应用时长排行改为更清晰的两列竖排。
- 托盘右键菜单第一项改为“交互节律”，保持打开窗口功能。
- 继续优化后台退出，减少退出时的系统卡顿。

## 1.3

定位：处理启动、后台和托盘稳定性。

- 优化启动阶段的窗口显示，减少白屏。
- 修复启动后窗口跟随鼠标、位置漂移等问题。
- 托盘中增加打开窗口、刷新数据、退出后台。
- 优化托盘退出流程，降低退出时冻结感。
- 增加开机自启和后台启动相关能力的基础。
- 检查运行占用、缓存量和后台行为，为分发测试做准备。

## 1.2

定位：形成“交互节律”的正式命名和第一版可分发形态。

- 项目正式命名为“交互节律”。
- 关闭按钮默认隐藏到托盘，完全退出放到托盘菜单。
- 更新桌面图标、应用内图标和托盘图标。
- 热力图改为今天加前五天，共 6 天展示。
- 热力图居中，应用窗口收窄并按内容重新排版。
- 应用时长排行做成两列展示。
- 优化无边框窗口、关闭、最小化和主题按钮。
- 打包输出 `交互节律.zip`。
