# 叩舷 v1.7.2 草稿

这一版是维护版草稿，目标是让真实使用里的启动、更新和发布包问题更容易排查。它不改变隐私边界，也不新增记录内容。

## 主要变化

- 新增本地诊断日志，记录启动、关闭、窗口、托盘、更新检查和更新安装准备阶段的关键事件。
- 更新安装脚本会写入 `last-update-state.json`，方便判断卡在数据备份、程序替换、启动新版还是回滚阶段。
- 更新失败时继续尝试恢复旧程序文件和用户数据。
- 新增 `scripts/preflight_release.py`，检查分享包、GitHub 包和更新清单。
- `build.ps1` 生成发布包后自动运行预检，避免数据库、日志、缓存和调试截图进入分享包。

## 验证重点

- 源码编译检查通过。
- 当前 `dist/releases` 里的 `叩舷.zip`、`interaction-rhythm.zip` 和 `update.json` 通过预检。
- 发布前仍需运行打包版冒烟测试：启动、托盘退出、热力图、导出、手感回放、设置和检查更新。

## 下载

正式发布后使用：

```text
interaction-rhythm-setup-v1.7.2.exe
```
